# MVP Acceptance Criteria (Real-Food)

## Product outcomes
1. User can complete first-session setup in under 3 minutes.
2. User can complete at least one full behavior loop:
   - Generate plan → check groceries → healthify craving.
3. Gamification feedback is visible, consistent, and tied to real actions.

## Must-have functional criteria
- Auth works for email/password and social demo flows.
- Onboarding captures flavor + dietary preferences and persists them.
- Home shows daily quest states and completion percentage.
- Quest completion grants XP and shows in-app reward feedback.
- Food Database search route loads and returns results list.
- TypeScript compile passes without errors.

## UX criteria
- UI style is modern and premium (clean spacing, clear hierarchy).
- Reward moments are subtle and non-intrusive (toast animation).
- Primary actions are obvious from each key screen.

## Release gate
MVP is considered ready for private beta when:
- All **Must-have** criteria pass.
- QA checklist manual happy-path checks are complete.
- No P0/P1 crashes or route blockers remain.
